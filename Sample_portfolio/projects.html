1. The Challenge

The client noticed a consistent drop in retained users but couldn't pinpoint the cause. The data was siloed across three different systems: CRM, Transaction Logs, and Support Tickets.

2. Data Processing (ETL)

I merged the datasets using SQL and utilized Python to clean missing values in the 'Last_Login' column.

# Imputing missing values based on session average df['last_login'].fillna(df['session_duration'].mean(), inplace=True) # Creating a churn flag (1 if inactive > 30 days) df['churned'] = df['days_since_login'].apply(lambda x: 1 if x > 30 else 0)

4. Strategic Recommendations

  • Implement a Grace Period: Extended the trial by 7 days to bridge the payment gap.
  • Triggered Emails: Set up automated alerts for users showing low activity on Day 85.
  • Outcome: These changes reduced the churn rate from 8% to 6.8% within one quarter.
